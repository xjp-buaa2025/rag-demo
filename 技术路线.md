## 二、模块1：数据预处理

### 1.1 维修手册预处理

```
输入：维修手册 PDF / Word 文档

Step1 格式转换
  PDF → 纯文本（pdfplumber / PyMuPDF）
  保留章节标题层级信息

Step2 文本分块（Chunking）
  策略：按装配步骤段落切分
  规则：
    - 检测"步骤X"/"第X步"/"X."等序号模式作为切分点
    - 每块控制在 400~600 字
    - 相邻块保留 50 字重叠（避免实体跨块丢失）

Step3 文本清洗
  - 去除页眉页脚、图表编号
  - 统一数字单位格式（N·m / N.m → N·m）
  - 繁体→简体转换（如适用）

输出：结构化文本块列表
  [{"chunk_id": "c001", "section": "3.2 HPC装配", "text": "..."}]
```

### 1.2 BOM表预处理

```
输入：平铺Excel表（零件号 | 名称 | 数量）

Step1 读取并标准化
  - 统一列名（part_number / name / quantity）
  - 去重、去空行

Step2 构建零件注册表（全局锚点）
  part_registry = {
    "PN-001": {"name": "高压压气机第3级转子叶片", "quantity": 36},
    "PN-002": {"name": "高压压气机外机匣",        "quantity": 1},
    ...
  }

Step3 名称规范化（为后续对齐准备）
  - 提取关键词：部件名 + 级次 + 类型
  - "高压压气机第3级转子叶片" → tokens: [高压压气机, 第3级, 转子, 叶片]

输出：part_registry.json
```

### 1.3 3D CAD模型预处理

```
输入：STEP格式 (.stp) 发动机装配模型

Step1 装配树解析（pythonocc / Open CASCADE）
  读取 STEP 文件中的产品层级结构
  输出：嵌套树形结构
  {
    "engine_assembly": {
      "HPC_Assembly": {
        "HPC_Rotor": {
          "HPC_Rotor_Blade_Stage3_001": {},
          "HPC_Rotor_Blade_Stage3_002": {},
          ...
        },
        "HPC_Outer_Casing": {}
      }
    }
  }

Step2 配合约束提取
  解析 STEP 中的 NEXT_ASSEMBLY_USAGE_OCCURRENCE 关系
  + 分析约束类型：同轴/贴合/平行/距离
  输出：约束列表
  [
    {"part_a": "HPC_Rotor_Blade_Stage3", "part_b": "HPC_Rotor_Disk",
     "constraint": "concentric", "interface": "榫头-榫槽"},
    ...
  ]

Step3 空间邻接分析
  基于包围盒（Bounding Box）重叠检测物理相邻零件
  阈值：间距 < 2mm 视为邻接
  输出：邻接矩阵

Step4 PMI标注提取（如有）
  提取嵌入模型的尺寸/公差标注
  输出：{"part": "xxx", "spec_type": "tolerance", "value": "±0.05mm"}

输出：cad_assembly_tree.json + cad_constraints.json + cad_adjacency.json
```

## 三、模块2：知识提取

### 2.1 LLM信息抽取（维修手册）

```
模型选择：GPT-4o / Qwen2.5-72B（中文优先）

Prompt设计（Few-shot + 领域约束）：

─────────────────────────────────────────
System Prompt:
你是航空发动机装配领域的知识抽取专家。
严格按照以下本体抽取实体和关系，不得添加本体之外的类型。

实体类型定义：
- Part(零件)：具体零部件，需有明确名称
- Assembly(组件)：由多个零件构成的子系统
- Procedure(工序)：具体操作动作
- Tool(工具)：装配所需工具器具
- Specification(规范)：数值型技术要求（扭矩/间隙/公差）
- Warning(注意事项)：安全警告或质量要求

关系类型定义：
- precedes：工序A必须在工序B之前完成
- participatesIn：零件参与某工序
- requires：工序需要某工具
- specifiedBy：工序/接口受某规范约束
- hasWarning：工序有某注意事项

Few-shot示例：（附3~5条人工精标样本）
─────────────────────────────────────────

输出格式（强制JSON）：
{
  "entities": [
    {"id": "e1", "type": "Procedure", "text": "插入叶片榫头"},
    {"id": "e2", "type": "Procedure", "text": "施加预紧扭矩"},
    {"id": "e3", "type": "Part",      "text": "高压压气机第3级叶片"},
    {"id": "e4", "type": "Tool",      "text": "扭力扳手"},
    {"id": "e5", "type": "Specification", "text": "50N·m"}
  ],
  "relations": [
    {"head": "e1", "tail": "e2", "type": "precedes"},
    {"head": "e3", "tail": "e1", "type": "participatesIn"},
    {"head": "e2", "tail": "e4", "type": "requires"},
    {"head": "e2", "tail": "e5", "type": "specifiedBy"}
  ]
}

后处理校验：
  ① 格式校验：JSON解析成功率监控
  ② DAG校验：precedes关系不得成环（装配序列必须为有向无环图）
  ③ 实体完整性：每个Procedure至少有1个precedes或participatesIn
  ④ 失败重试：校验不通过则携带错误信息重新请求LLM
```

### 2.2 BOM结构化解析（规则方法）

```python
# 直接规则映射，无需模型
import pandas as pd

def parse_bom(excel_path):
    df = pd.read_excel(excel_path)
    part_registry = {}
    for _, row in df.iterrows():
        part_registry[row['零件号']] = {
            'name':     row['名称'],
            'quantity': row['数量']
        }
    return part_registry
```

### 2.3 CAD几何知识转三元组

```
装配树 → isPartOf三元组
  (HPC_Rotor_Blade_Stage3, isPartOf, HPC_Rotor)
  (HPC_Rotor, isPartOf, HPC_Assembly)

配合约束 → matesWith三元组
  (HPC_Rotor_Blade_Stage3, matesWith, HPC_Rotor_Disk)
  附属性：constraint_type=concentric, interface=榫头-榫槽

邻接关系 → adjacentTo三元组
  (HPC_Rotor_Blade_Stage3, adjacentTo, HPC_Stator_Vane_Stage3)

PMI → Specification节点
  (HPC_Rotor_Blade_Stage3, specifiedBy, Spec_clearance_0.05mm)
```

## 四、模块3：跨模态实体对齐

```
目标：将 CAD组件名 / 手册文本提及 统一映射到 BOM零件号

─────────────────────────────────────────────────────
第一级：规则对齐（预计覆盖70%）

  航空缩写词典：
  {HPC→高压压气机, LPC→低压压气机, HPT→高压涡轮,
   LPT→低压涡轮, Blade→叶片, Disk→叶盘, Casing→机匣,
   Rotor→转子, Stator→静子, Vane→导向叶片,
   Stage/S→第X级, Bearing→轴承, Seal→封严环, ...}

  处理流程：
  "HPC_Rotor_Blade_Stage3_001"
   → 分词: [HPC, Rotor, Blade, Stage3]
   → 扩展: [高压压气机, 转子, 叶片, 第3级]
   → 重组: "高压压气机转子叶片第3级"
   → 精确匹配BOM → 命中PN-001 ✓

─────────────────────────────────────────────────────
第二级：字符串模糊匹配（预计覆盖20%）

  算法：Jaccard字符相似度
  score = |A∩B| / |A∪B|  （基于字符集合）

  score > 0.85 → 自动对齐
  score 0.70~0.85 → 标记为候选，写入待审列表
  score < 0.70 → 进入第三级

─────────────────────────────────────────────────────
第三级：语义向量匹配（预计覆盖9%）

  模型：BERT-Chinese（或text2vec-base-chinese）
  将CAD名称（扩展后）和BOM名称分别编码为向量
  取余弦相似度最高的BOM条目

  + 位置辅助验证：
  CAD树中第N层的节点 与 同类零件的BOM数量级相符 → 置信度+0.1

─────────────────────────────────────────────────────
未对齐处理：

  CAD有、BOM无 → 标准件（螺栓/垫片）→ 新建节点，来源=CAD_only
  三级均失败   → 写入 unaligned_log.json，作为论文局限性

─────────────────────────────────────────────────────
输出：alignment_table.json
  {
    "HPC_Rotor_Blade_Stage3_001": {
      "bom_id": "PN-001",
      "confidence": 0.94,
      "method": "rule"
    },
    ...
  }
```

## 五、模块4：知识图谱构建与存储

### 5.1 KG Schema定义

```
节点类型（7类）：
  (:Part)         {id, name, material, quantity}
  (:Assembly)     {id, name, level, function}
  (:Procedure)    {id, description, seq_no}
  (:Tool)         {id, name, type}
  (:Specification){id, spec_type, value, unit}
  (:Interface)    {id, interface_type}
  (:GeoConstraint){id, constraint_type}

边类型（9类）：
  [:isPartOf]        Part/Assembly → Assembly    （来源：BOM+CAD）
  [:precedes]        Procedure → Procedure       （来源：手册）
  [:participatesIn]  Part → Procedure            （来源：手册）
  [:requires]        Procedure → Tool            （来源：手册）
  [:specifiedBy]     Procedure/Interface → Spec  （来源：手册+CAD）
  [:matesWith]       Part → Part                 （来源：CAD）
  [:adjacentTo]      Part → Part                 （来源：CAD）
  [:hasInterface]    Part → Interface            （来源：CAD）
  [:constrainedBy]   Interface → GeoConstraint   （来源：CAD）
```

### 5.2 三元组融合与存储

```
Step1 实体去重
  以BOM零件号为主键，合并手册和CAD中的同一实体

Step2 关系合并
  同源同类型的重复关系取并集

Step3 冲突消解
  规范值冲突（手册50N·m vs CAD标注52N·m）→ 以手册为准，记录冲突日志

Step4 完整性校验
  - precedes链必须构成DAG（检测环路）
  - 孤立节点（无任何关系）标记为待补充

Step5 写入Neo4j
  Cypher批量导入（每批1000条三元组）

输出：Neo4j图数据库
  节点数：~XXXX个
  关系数：~XXXX条
```

## 六、模块5：HybridRAG检索

### 6.1 三路检索

```
用户查询Q: "装配HPC第3级叶片需要哪些步骤和工具？"
                    │
          ┌─────────┼──────────┐
          ▼         ▼          ▼

【图检索】          【向量检索】      【BM25检索】
CoE Planner-Agent   FAISS索引        关键词倒排索引
    │               │               │
    ▼               ▼               ▼
图路径结果R_G    语义相似块R_V    关键词匹配块R_B
    │               │               │
    └───────────────┼───────────────┘
                    │
                    ▼
             【Re-ranking重排序】
             Cross-Encoder重新打分
             考虑：语义相关性 + 工序顺序完整性
                    │
                    ▼
             Top-K最终上下文

图检索（CoE）特别说明：
  - 基于precedes关系链，确保检索到完整的工序序列
  - 不破坏工序顺序（不允许返回乱序步骤）
  - 通过matesWith关系扩展相关零件信息
```

### 6.2 序列感知检索（核心创新）

```
普通RAG问题：
  可能返回 Step3、Step1、Step7（无序）→ 装配指导混乱

本文解决方案：
  检索结果中的Procedure节点按precedes链拓扑排序后输出
  保证：Step1 → Step2 → Step3 → ... 严格有序

算法：
  1. 从检索结果中识别所有Procedure节点
  2. 在KG中查询这些节点间的precedes关系
  3. 拓扑排序（Kahn算法）得到正确顺序
  4. 按序组装最终上下文
```

## 七、模块6：装配引导Agent输出

```
输入：Top-K有序上下文 + 用户查询

LLM生成最终回答，格式：

─────────────────────────────────────
【装配任务】高压压气机第3级叶片装配

【准备工具】
  - 扭力扳手（量程0~100N·m）
  - T-205锁紧工具

【装配步骤】
  Step1. 检查叶片榫头表面，确认无毛刺和损伤
         ⚠ 注意：禁止用手直接接触叶片型面
  Step2. 将叶片榫头对准叶盘榫槽，缓慢插入至底
         配合类型：间隙配合，配合面涂MoS₂润滑脂
  Step3. 使用扭力扳手施加扭矩至 50N·m
         ⚠ 注意：分三次施加，不得一次到位
  Step4. 用T-205工具锁紧锁片，确认锁到位

【技术规范】
  - 装配扭矩：50N·m ±2N·m
  - 叶片-叶盘间隙：0.05~0.12mm
─────────────────────────────────────

支持多轮对话：
  用户追问 → Agent基于当前上下文 + KG继续检索补充信息
```

## 八、实验设计

### 8.1 评估指标

```
KG构建质量：
  - 实体对齐准确率（人工验证200条）
  - 关系抽取F1（Precision / Recall / F1）
  - KG完整性（DAG合法率）

检索性能：
  - Recall@K（检索结果包含正确答案的比例）
  - MRR（平均倒数排名）
  - 序列正确率（返回步骤的顺序准确度）← 本文新指标

问答质量：
  - 准确率（Accuracy）
  - F1 Score
  - 幻觉率（Hallucination Rate）
  - 工序完整率（返回的步骤覆盖率）← 本文新指标
```
